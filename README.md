"# CornerGrocerProject"



**Project Summary**



&#x09;For my portfolio submission, I selected my Corner Grocer project. The program reads grocery items from an input file, counts how many times each item appears, and allows the user to search for an item, display all item frequencies, or view a histogram. This project helped me practice object-oriented programming, file input and output, and data organization using C++.



**What I Did Particularly Well**



&#x09;I organized my program by creating a GroceryTracker class that kept the data and functions together. I also added input validation to prevent invalid menu choices from causing errors. Using clear variable names, comments, and consistent formatting made the code easy to read and understand.



**Where I Could Enhance My Code**



&#x09;I could improve the program by allowing the user to choose the input file instead of using a hardcoded filename. I would also make item searches case-insensitive so users would not have to enter the exact capitalization. These improvements would make the program more user-friendly, flexible, and efficient.



**Challenges and Resources**



&#x09;The most challenging part of the project was learning how to use a map to store and count grocery items correctly. I overcame this challenge by reviewing the course materials, testing my program with different inputs, and debugging my code. I also used C++ documentation and examples to better understand file input and the Standard Library.



**Transferable Skills**



&#x09;This project strengthened my skills in object-oriented programming, file handling, and using maps to organize data. I also improved my debugging, problem-solving, and input validation skills while developing the program. These skills will be useful in future programming courses and real-world software development projects.



**Maintainability, Readability, and Adaptability**



&#x09;I made the program maintainable by separating the data and functions into a dedicated class. I used descriptive function names, meaningful variable names, and comments to improve readability. The program is also adaptable because new features can be added without making major changes to the existing code.
